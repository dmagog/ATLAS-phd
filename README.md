# ATLAS Agent

**ATLAS (Assistant for Technical Learning & Attestation Support)** — агентный ассистент для подготовки к кандидатскому минимуму по технической специальности.

## Проблема

Подготовка к кандидатскому минимуму требует работы с большим объемом разрозненных материалов (учебники, статьи, конспекты), а также регулярной самопроверки.

- Долго искать релевантные фрагменты вручную.
- Трудно проверять достоверность ответов и не "уходить" в галлюцинации.
- Нет быстрого цикла "изучил тему -> проверил себя -> получил обратную связь".

## Аудитория

- Аспиранты и соискатели по техническим направлениям.
- Пользователи, которым нужен ассистент по подготовке к техническому экзамену с опорой на источники и формулы.

## Что сделает PoC на демо

PoC демонстрирует агентный контур поверх RAG:

1. `Planner Agent` классифицирует запрос (`Q&A`, `self-check`, уточнение).
2. `Retrieval Tool` подбирает релевантные фрагменты из базы знаний.
3. `Answer Agent` формирует ответ только по найденным источникам.
4. `Verifier Agent` проверяет достаточность evidence и применяет guardrails.
5. В режиме `self-check` система генерирует вопросы, оценивает ответы и дает рекомендации.

Каналы MVP: web-чат как приоритетный интерфейс для демо и работы с формулами, Telegram-бот как дополнительный канал, запуск локально.

## Что PoC НЕ делает (out-of-scope)

- Не покрывает все дисциплины и все форматы документов (например, `djvu`).
- Не обещает production-SLA и высокую нагрузку.
- Не заменяет преподавателя и не принимает экзаменационные решения.
- Не строит продвинутую долгосрочную learning-аналитику.
- Не выполняет внешние действия от лица пользователя (письма, платежи, изменения внешних систем).

## План локального запуска MVP (целевой runbook)

После добавления реализации MVP запуск выполняется по шагам:

1. Подготовить окружение (`Python`, `Docker`, `docker-compose`).
2. Заполнить `.env` (LLM API key, DB settings, `Telegram token` при включении бота).
3. Поднять инфраструктуру (`docker compose up -d`).
4. Применить миграции БД.
5. Запустить API/web слой как основной интерфейс MVP и Telegram-бот как дополнительный канал.
6. Прогнать smoke-check сценарии из `docs/acceptance_tests.md` (`AT-01`, `AT-03`, `AT-06`, `AT-14`).
