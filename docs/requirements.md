# Матрица требований и трассировка v1.1 (MVP)

Примечание:
- В матрице часть требований декомпозирована для удобства трассировки (например, отдельные строки по каналам и агентным узлам).
- Приоритет проверки — смысловая трассировка `ТЗ -> UC -> AT`, а не буквальное совпадение нумерации во всех документах.

## 3. KPI/SLO (измеримые цели)

| ID | Показатель | Цель |
|---|---|---|
| KPI-R1 | Доля неотказных ответов с валидными источниками | >= 95% |
| KPI-R2 | Корректные отказы на контрольном наборе | >= 85% |
| KPI-R3 | Галлюцинации на gold-set | <= 10% |
| KPI-S1 | MAE автооценки открытых ответов против эксперта | <= 1.0 |
| KPI-S2 | Доля непустой осмысленной обратной связи | >= 95% |
| SLO-O1 | p95 latency Q&A | <= 15 сек |
| SLO-O2 | Доля технических ошибок | <= 3% |
| SLO-O3 | Успешность ingestion-job | >= 95% |
| KPI-A1 | Точность роутинга Planner | >= 90% |
| KPI-A2 | Корректные блокировки Verifier (hard-gate) | >= 85% |
| KPI-A3 | Доля валидных self-check оценок (без invalid_evaluation) | >= 95% |

## 4. Явно исключено из MVP

| ID | Ограничение | Обоснование |
|---|---|---|
| OUT-01 | Нативная поддержка `djvu` | Снижение сложности ingestion в пилоте |
| OUT-02 | Режим преподавателя | Вынесен в roadmap |
| OUT-03 | High-load и production-SLA | Не требуется для локального MVP |
| OUT-04 | Продвинутые дашборды длительного прогресса | Не критично для пилота |
| OUT-05 | Кросс-сессионная персонализация Q&A | В MVP используется сессионная память без долгосрочной адаптации |
| OUT-06 | Telegram-бот | Отложен за рамки текущего MVP; основной канал — web |
