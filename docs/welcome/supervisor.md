# Welcome — научрук

Вы видите аналитику по подготовке группы аспирантов к кандэкзамену. Цель — найти **слабые темы** программы, увидеть, **где аспиранты буксуют**, и направить семинары/консультации в нужное русло.

ATLAS — обучающая среда, а не аттестационная: ваши действия здесь не выставляют формальных оценок. Реальная сдача экзамена и аттестация — вне системы.

## Регистрация

Tenant-admin (методист кафедры) отправит инвайт-ссылку:

```
https://atlas.ваша-кафедра/invites/<code>
```

Зайдите, заполните email/пароль, подтвердите согласие. После регистрации сразу залогинены.

## Что доступно

### 1. Heatmap программы
`GET /tenants/optics-kafedra/supervisor/heatmap` — общегрупповая статистика по темам:
- per-topic: общее число попыток, число «провалов» (score < 3), доля провалов с 95% Wilson CI
- сортировка по убыванию доли провалов — самые проблемные темы первыми

Если в группе **меньше 5 студентов** или **меньше 30 завершённых попыток**, heatmap не показывается (BDD 5.3 + 5.6 — статистика становится надёжной только на достаточной выборке). В таком случае API возвращает `is_below_threshold: true` с причиной — это сигнал «дайте группе пожить с системой ещё неделю».

### 2. Drilldown по проблемному топику
`GET /tenants/.../supervisor/topics/{topic_id}/drilldown` — подробности по конкретному билету:
- сколько попыток ниже threshold
- какие error_tags часто всплывают (incomplete, logic_gap, terminology, factual_error)

Дальше — это вход в семинар: «по топику „Дифракция Френеля" 8 из 10 завалились на logic_gap → разбираем зоны Френеля на следующем семинаре».

### 3. Список аспирантов
`GET /tenants/.../supervisor/students` — список группы.

Видимость:
- Аспиранты с **opt-in** показаны с email и student_id — на них можно открыть профиль.
- Аспиранты **без opt-in** показаны как `Аспирант #N` без id — их индивидуальный профиль вам недоступен.

Это **сознательное privacy-решение**, не баг. Если аспирант не дал согласие на персональную видимость — вы видите его вклад в общегрупповую статистику, но не его индивидуальные результаты.

### 4. Профиль студента (только при opt-in)
`GET /tenants/.../supervisor/students/{student_id}/profile`:
- общее число попыток
- топик-уровень: последний балл, дата, общее количество попыток

Каждый просмотр профиля **логируется в audit_log** (`personal_data.access`). Это видимый для аспиранта след — он может посмотреть в свой `/me` и увидеть, кто и когда читал его данные.

Попытка открыть профиль не-opted-in аспиранта вернёт **404 «Student not found»** (мы не различаем «не существует» и «не дал согласие» — иначе утекает информация). При этом запись `privacy.violation_attempt` всё равно делается в audit-log: tenant-admin это видит.

## Limits

- Вы видите только **свой тенант** (свою кафедру). Cross-tenant запросы отфутболиваются 403.
- Вы **не можете** управлять корпусом, программой или приглашениями — это полномочия tenant-admin'а.
- Вы **не можете** посмотреть профили без opt-in — даже если сегодня смотрели, и аспирант снял opt-in, новые запросы вернут 404.

## Этика

- Ничто из того, что вы видите, **не выставляет аттестационных решений**. Heatmap — это «погодная карта» подготовки группы, не журнал успеваемости.
- Privacy default — анонимные агрегаты. Если хотите, чтобы аспирант открыл вам персональную видимость — попросите его сделать opt-in лично, не давите через систему.
- При просадке какой-то темы **сначала** обсудите с группой что не зашло, **потом** меняйте план семинаров.
